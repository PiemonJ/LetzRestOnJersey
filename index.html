Letz REST on Jersey :

Sample Tutorial/ sample code for Using Jersey in building the REST services, following architectural standards, writing unit test cases using jersey Test framework for REST service Resources, implementing excepting mapping for making the code robust, hateoas support in building the REST services with Atom links.

Tools Used are:

jersey(1.17), spring, maven

Highlights: (use this instead of Samples)

  1. Jersey made easy to understand with sample standard REST service

  2. Jersey-Spring integration

  3. POJO mapping feature of jersey (converting java object to the JSON)

  4. Jersey, spring with maven enabled

  5. Test cases for unit testing Resources using Jersey Test framework to keep the code reliable

  6. Exception handling via exception mappers to keep code robus

  7. HateOAS support via annotation (Newly introduced and interesting part.
